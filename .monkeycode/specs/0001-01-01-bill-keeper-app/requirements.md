# 票夹管家 App 需求文档

## 1. 简介

### 1.1 项目概述

**项目名称：** 票夹管家  
**项目类型：** 个人票据收藏移动应用  
**核心功能：** 拍照存档证件、门票、车票、发票等各类票据，通过 OCR 自动提取信息并智能分类，方便日后查找与回忆。

### 1.2 设计原则

- **隐私优先：** OCR 识别由用户手动点击触发，绝不自动进行；数据默认本地存储。
- **智能 OCR：** 使用本地 RapidOCR（模型静默下载，精度优化）。
- **简洁界面：** 无底部导航，所有次要功能收于右上角折叠菜单。

### 1.3 四阶段概述

| 阶段 | 名称 | 核心任务 |
|------|------|----------|
| 第一阶段 | 基础框架与用户系统 | 搭建应用骨架，实现用户登录、数据模型、首页布局及折叠菜单 |
| 第二阶段 | 核心票据管理与扫描矫正 | 实现票据上传、图片扫描矫正、OCR 识别、自动标签推荐 |
| 第三阶段 | 搜索与管理 | 实现搜索页、标签/合集管理、回收站 |
| 第四阶段 | 测试与基础发布 | 全面测试、性能优化、打包上架 |

---

## 2. 术语表

| 术语 | 定义 |
|------|------|
| **票据** | 用户拍照存档的证件、门票、车票、发票等各类凭证 |
| **OCR** | Optical Character Recognition，光学字符识别，用于提取图片中的文字 |
| **RapidOCR** | 本地 OCR 识别引擎，支持离线使用 |
| **扫描矫正** | 自动检测票据边缘、透视校正、裁剪、旋转拉直的图像处理流程 |
| **合集** | 用户创建的票据分组，用于组织和管理多个相关票据 |
| **标签** | 用户为票据添加的分类关键词，支持筛选和搜索 |
| **回收站** | 临时存储已删除票据的空间，支持 30 天内恢复 |
| **批量操作** | 用户同时对多个票据执行同一操作（如批量删除、批量移动） |

## 3. 第一阶段需求：基础框架与用户系统

### 3.1 数据模型

#### REQ-001：用户数据模型

**用户故事：** 作为系统，我需要管理用户账户，以便为用户提供个性化的票据存储服务。

**验收标准：**

1. WHEN 用户首次打开应用并完成手机号验证注册，the system SHALL 创建用户账户，包含手机号和初始存储容量（总容量 1GB，已用容量 0）。
2. WHILE 系统需要显示用户信息，the system SHALL 提供用户手机号和存储用量的查询接口。
3. AS 系统计算存储用量，the system SHALL 统计用户所有图片及元数据的大小总和。

#### REQ-002：票据数据模型

**用户故事：** 作为系统，我需要管理票据数据，以便用户存储和查找各类票据。

**验收标准：**

1. WHEN 用户创建新票据时，the system SHALL 记录以下字段：标题、OCR 全文、标签列表、合集 ID、位置、备注、创建时间。
2. AS 系统查询票据列表，the system SHALL 返回票据的缩略图、标题、日期信息。
3. WHEN 用户按标签或合集筛选时，the system SHALL 仅返回匹配的票据。

#### REQ-003：图片数据模型

**用户故事：** 作为系统，我需要管理票据的图片附件，以便展示和存储原始图片。

**验收标准：**

1. WHEN 用户上传图片时，the system SHALL 记录：所属票据 ID、本地路径、缩略图路径、排序序号。
2. WHEN 删除票据时，the system SHALL 同时删除所有关联的图片文件。

#### REQ-004：合集数据模型

**用户故事：** 作为系统，我需要管理合集数据，以便用户组织和分类票据。

**验收标准：**

1. WHEN 用户创建合集时，the system SHALL 记录：合集名称、所属用户 ID、创建时间。
2. WHEN 删除合集时，the system SHALL 将对应票据的合集 ID 置空。

#### REQ-005：标签数据模型

**用户故事：** 作为系统，我需要管理标签数据，以便用户分类票据。

**验收标准：**

1. WHEN 用户创建标签时，the system SHALL 记录：标签名称、所属用户 ID、创建时间。
2. THE system SHALL 为每个标签分配唯一标识符。

#### REQ-006：回收站数据模型

**用户故事：** 作为系统，我需要管理回收站数据，以便支持票据恢复和彻底删除。

**验收标准：**

1. WHEN 用户删除票据时，the system SHALL 将票据移入回收站，记录票据 ID 和删除时间。
2. IF 回收站中的票据超过 30 天，the system SHALL 自动彻底删除该票据及其所有关联图片。
3. WHEN 用户恢复票据时，the system SHALL 将票据从回收站移回正常列表。

#### REQ-007：用户-票据关联

**用户故事：** 作为系统，我需要管理用户与票据的关联关系，以便隔离不同用户的数据。

**验收标准：**

1. THE system SHALL 确保每个票据仅属于一个用户。
2. THE system SHALL 仅允许用户访问自己的票据数据。

### 3.2 用户登录

#### REQ-010：手机号验证码登录

**用户故事：** 作为新用户，我想要通过手机号注册登录，以便安全地使用应用。

**验收标准：**

1. WHEN 用户输入手机号并点击获取验证码，the system SHALL 向该手机号发送 6 位数字验证码。
2. THE system SHALL 确保验证码在 5 分钟内有效。
3. THE system SHALL 限制同一手机号在 24 小时内最多发送 10 次验证码。
4. IF 用户连续 5 次输入错误验证码，the system SHALL 将该验证码视为失效，要求用户重新获取。
5. WHEN 用户输入正确验证码并提交，the system SHALL 创建用户账户并登录。
6. WHEN 用户已存在账户并完成验证，the system SHALL 直接登录并恢复会话状态。
7. WHEN 登录成功后，the system SHALL 在本地存储登录状态。

#### REQ-011：登录状态管理

**用户故事：** 作为系统，我需要管理登录状态，以便用户无需重复登录。

**验收标准：**

1. Upon 用户成功登录，the system SHALL 将登录状态持久化到本地存储。
2. WHEN 用户重新打开应用，the system SHALL 自动恢复登录状态。
3. WHEN 用户退出登录时，the system SHALL 清除本地登录状态。

#### REQ-012：登录安全

**用户故事：** 作为系统，我需要保护用户账户安全，防止恶意注册和暴力破解。

**验收标准：**

1. THE system SHALL 验证手机号格式（+86 开头的 11 位数字）。
2. THE system SHALL 限制单个设备在 24 小时内尝试登录最多 20 次。
3. IF 检测到异常登录行为，the system SHALL 临时冻结该手机号的登录尝试。

### 3.3 首页布局

#### REQ-020：顶部栏布局

**用户故事：** 作为用户，我想要在首页快速访问个人中心和主要功能。

**验收标准：**

1. WHEN 用户打开应用首页，the system SHALL 显示顶部栏，包含左侧头像、中间应用名称"票夹管家"、右侧折叠菜单图标（≡）。
2. WHEN 用户点击头像，the system SHALL 跳转到个人中心页面。
3. WHEN 用户点击折叠菜单图标，the system SHALL 显示折叠菜单面板。

#### REQ-021：折叠菜单

**用户故事：** 作为用户，我想要通过折叠菜单访问次要功能。

**验收标准：**

1. WHEN 用户点击折叠菜单图标，the system SHALL 显示以下菜单项：回收站、标签管理、合集管理、设置、帮助反馈。
2. WHEN 用户点击菜单项，the system SHALL 跳转到对应页面并关闭菜单。

#### REQ-022：搜索框

**用户故事：** 作为用户，我想要在首页快速搜索票据。

**验收标准：**

1. WHEN 用户打开应用首页，the system SHALL 在顶部栏下方显示搜索框。
2. WHEN 用户点击搜索框，the system SHALL 跳转到搜索页面。

#### REQ-023：快速上传卡片

**用户故事：** 作为用户，我想要快速上传新票据。

**验收标准：**

1. WHEN 用户打开应用首页，the system SHALL 显示快速上传卡片，包含"拍照"和"从相册选择"两个按钮。
2. WHEN 用户点击"拍照"，the system SHALL 调用相机进行拍摄。
3. WHEN 用户点击"从相册选择"，the system SHALL 打开系统相册供选择。

#### REQ-024：快速筛选

**用户故事：** 作为用户，我想要快速筛选特定标签或合集的票据。

**验收标准：**

1. WHEN 用户打开应用首页，the system SHALL 在上传卡片下方显示一行下拉菜单，用于筛选标签和合集。
2. WHEN 用户选择标签或合集时，the system SHALL 动态加载匹配的票据列表。
3. WHEN 用户点击"清除筛选"按钮，the system SHALL 重置筛选条件并显示全部票据。

#### REQ-025：票据列表

**用户故事：** 作为用户，我想要查看我的所有票据。

**验收标准：**

1. WHEN 用户打开应用首页，the system SHALL 以网格或列表形式展示票据，显示缩略图、标题、日期。
2. WHEN 用户下拉列表时，the system SHALL 刷新票据列表。
3. WHEN 用户上拉列表时，the system SHALL 加载更多票据（分页加载，每页 20 条）。

#### REQ-026：批量操作

**用户故事：** 作为用户，我想要对多个票据进行批量操作，提高管理效率。

**验收标准：**

1. WHEN 用户长按票据卡片时，the system SHALL 进入批量选择模式。
2. IN 批量选择模式，the system SHALL 显示已选数量和批量操作按钮。
3. THE system SHALL 提供批量删除、批量移动到合集、批量添加标签操作。
4. WHEN 用户点击"全选"，the system SHALL 选择当前筛选条件下所有票据。

**验收标准：**

#### REQ-033：退出登录

**验收标准：**

## 4. 第二阶段需求：核心票据管理与扫描矫正

### 4.1 上传流程

#### REQ-050：图片选择

**用户故事：** 作为用户，我想要选择多张图片作为票据附件。

**验收标准：**

1. WHEN 用户点击首页的"拍照"或"从相册选择"，the system SHALL 进入图片选择页面。
2. THE system SHALL 支持单选和多选模式。
3. WHEN 用户选择图片时，the system SHALL 限制最多选择 9 张图片。
4. WHEN 用户完成选择并确认，the system SHALL 跳转到图片编辑页面。

#### REQ-051：图片编辑页面布局

**用户故事：** 作为用户，我想要在编辑页面管理票据信息和图片。

**验收标准：**

1. WHEN 用户进入图片编辑页面，the system SHALL 显示标题输入框（可留空）。
2. THE system SHALL 限制标题长度在 0-100 个字符之间。
3. WHEN 用户进入图片编辑页面，the system SHALL 显示图片预览区，支持左右滑动，每张图片可删除。
4. WHEN 用户点击图片下方的"扫描矫正"按钮，the system SHALL 调用图像处理插件进行边缘检测、透视校正、裁剪、旋转拉直，处理后替换原图。

#### REQ-052：OCR 识别（手动触发）

**用户故事：** 作为用户，我想要手动触发 OCR 识别提取文字。

**验收标准：**

1. WHEN 用户点击"识别文字"按钮，the system SHALL 开始本地 OCR 识别。
2. IF OCR 模型未下载，the system SHALL 后台静默下载模型后进行识别，并显示下载进度。
3. IF OCR 模型已下载，the system SHALL 立即开始识别并显示加载动画。
4. THE system SHALL 确保 OCR 识别在 3 秒内完成（正常网络条件下）。
5. WHEN OCR 识别完成时，the system SHALL 显示识别结果，并自动生成推荐标签。
6. IF OCR 识别失败，the system SHALL 显示错误信息，显示"识别失败，请重试"提示，并保留用户可手动输入文字的入口。

#### REQ-053：OCR 结果处理

**用户故事：** 作为用户，我想要编辑 OCR 识别结果和标签。

**验收标准：**

1. WHEN OCR 识别完成后，the system SHALL 在识别结果区域显示 OCR 全文，支持用户编辑。
2. WHEN OCR 识别完成后，the system SHALL 显示推荐标签区域，根据识别内容自动生成 3-5 个标签建议（如"证件""景点""车票"），用户点击即可添加。
3. WHEN 用户添加标签时，the system SHALL 在已选标签区域显示已选标签，用户可删除。
4. WHEN 用户需要添加自定义标签时，the system SHALL 提供"添加标签"输入框及按钮。

#### REQ-054：票据其他信息

**用户故事：** 作为用户，我想要设置票据的其他信息。

**验收标准：**

1. WHEN 用户编辑票据时，the system SHALL 提供合集选择下拉框，支持选择已有合集或新建合集。
2. WHEN 用户编辑票据时，the system SHALL 自动从图片 Exif 信息提取位置（GPS 坐标或地址文字），若无法提取则显示"未知位置"。
3. THE system SHALL 支持用户手动修改位置信息。
4. WHEN 用户编辑票据时，the system SHALL 提供备注输入框（多行文本），限制最多 500 个字符。
5. WHEN 用户点击底部"保存"按钮，the system SHALL 将票据存入数据库并跳转到详情页。

### 4.2 票据详情页

#### REQ-060：详情页布局

**用户故事：** 作为用户，我想要查看票据的完整信息。

**验收标准：**

1. WHEN 用户进入票据详情页，the system SHALL 在顶部显示标题、编辑按钮、更多操作按钮（分享、删除）。
2. WHEN 用户进入票据详情页，the system SHALL 显示图片轮播区域。
3. WHEN 用户进入票据详情页，the system SHALL 显示信息卡片，包含标签（可点击筛选）、合集、位置、备注、OCR 全文（可折叠，默认折叠超过 200 字符的内容）。

#### REQ-061：图片操作

**用户故事：** 作为用户，我想要对票据图片进行操作。

**验收标准：**

1. WHEN 用户点击图片轮播中的图片时，the system SHALL 放大显示图片，支持双指缩放。
2. WHEN 用户长按图片时，the system SHALL 弹出操作菜单，包含"扫描矫正"、"旋转（左转 90 度）"、"删除"选项。
3. WHEN 用户点击"扫描矫正"，the system SHALL 对当前图片进行扫描矫正并替换原图。
4. WHEN 用户点击"旋转"，the system SHALL 将图片左转 90 度并保存。
5. WHEN 用户点击"删除"，IF 票据有多张图片，the system SHALL 删除当前图片；IF 票据只有一张图片，the system SHALL 删除整个票据并移入回收站。

#### REQ-062：分享功能

**用户故事：** 作为用户，我想要分享票据给他人。

**验收标准：**

1. WHEN 用户点击"分享"按钮，the system SHALL 显示分享选项菜单。
2. THE system SHALL 支持分享为图片（当前展示的图片）或链接（应用内打开的 deep link）。
3. WHEN 用户选择"分享为图片"，the system SHALL 将当前显示的图片和票据标题生成图片并调用系统分享。
4. WHEN 用户选择"分享为链接"，the system SHALL 生成包含票据 ID 的短链接并调用系统分享。

#### REQ-063：票据操作

**用户故事：** 作为用户，我想要对票据进行管理操作。

**验收标准：**

1. WHEN 用户在详情页点击"添加到合集"，the system SHALL 显示合集选择列表。
2. IF 用户将票据从某个合集移出，the system SHALL 将该票据的合集 ID 置空。
3. WHEN 用户在详情页点击"删除"，the system SHALL 将票据移入回收站并显示"已移入回收站，可在 30 天内恢复"提示。

---

### 4.3 票据编辑页

#### REQ-070：编辑页功能

**用户故事：** 作为用户，我想要编辑已有的票据。

**验收标准：**

1. WHEN 用户从详情页点击编辑按钮，the system SHALL 加载已有数据到编辑页面。
2. WHEN 用户编辑票据时，the system SHALL 支持修改标题、OCR 全文、标签、合集、位置、备注。
3. WHEN 用户编辑票据时，the system SHALL 支持新增图片（拍照/相册）、删除图片、对单张图片进行扫描矫正。
4. WHEN 用户再次点击"识别文字"，the system SHALL 更新 OCR 全文与推荐标签。
5. WHEN 用户点击"保存"，the system SHALL 更新票据数据并返回详情页。

---

## 5. 第三阶段需求：搜索与管理

### 5.1 搜索页

#### REQ-080：搜索功能

**用户故事：** 作为用户，我想要搜索特定的票据。

**验收标准：**

1. WHEN 用户进入搜索页，the system SHALL 在顶部显示搜索框和"搜索"按钮。
2. WHEN 用户输入关键词并点击"搜索"，the system SHALL 搜索票据的标题和 OCR 全文，返回匹配结果。
3. THE system SHALL 支持关键词高亮显示。
4. WHEN 用户设置筛选条件时，the system SHALL 支持按标签筛选（多选）、按合集筛选（单选）、按日期范围筛选（开始日期-结束日期）。
5. WHEN 用户选择排序方式时，the system SHALL 支持时间倒序（默认）和相关度排序。

#### REQ-081：搜索历史

**用户故事：** 作为用户，我想要快速重复之前的搜索。

**验收标准：**

1. THE system SHALL 记录用户最近 10 条搜索关键词。
2. WHEN 用户点击搜索框，the system SHALL 显示搜索历史列表。
3. WHEN 用户点击历史记录，the system SHALL 自动填充关键词并搜索。
4. WHEN 用户点击"清除历史"，the system SHALL 删除所有搜索历史。

---

### 5.2 标签管理

#### REQ-090：标签管理功能

**用户故事：** 作为用户，我想要管理和优化我的标签。

**验收标准：**

1. WHEN 用户进入标签管理页，the system SHALL 列表显示所有标签及对应的票据数量，按使用频率降序排列。
2. WHEN 用户点击标签的"编辑"按钮，the system SHALL 显示重命名对话框。
3. THE system SHALL 限制标签名称在 1-20 个字符之间。
4. WHEN 用户点击标签的"合并"按钮，the system SHALL 显示标签选择列表，IF 用户确认合并，the system SHALL 将源标签合并到目标标签，更新所有相关票据的标签。
5. WHEN 用户点击标签的"删除"按钮，the system SHALL 显示确认对话框，IF 用户确认，the system SHALL 从所有票据中移除该标签并删除标签记录。

---

### 5.3 合集管理

#### REQ-100：合集管理功能

**用户故事：** 作为用户，我想要管理我的合集。

**验收标准：**

1. WHEN 用户进入合集管理页，the system SHALL 列表显示所有合集，显示每个合集的票据数量。
2. WHEN 用户点击"新建"按钮，the system SHALL 显示新建合集对话框，IF 用户输入合集名称并确认，the system SHALL 创建新合集。
3. THE system SHALL 限制合集名称在 1-30 个字符之间。
4. WHEN 用户点击合集的"编辑"按钮，the system SHALL 显示重命名对话框。
5. WHEN 用户点击合集的"删除"按钮，the system SHALL 显示确认对话框，IF 用户确认，the system SHALL 删除合集并将对应票据的合集 ID 置空。

---

### 5.4 回收站

#### REQ-110：回收站功能

**用户故事：** 作为用户，我想要恢复误删的票据或彻底删除。

**验收标准：**

1. WHEN 用户进入回收站页，the system SHALL 列表显示 30 天内删除的票据，显示剩余可恢复天数。
2. THE system SHALL 支持单选和多选操作。
3. WHEN 用户选择票据并点击"恢复"，the system SHALL 将票据恢复到原列表，并显示"已恢复"提示。
4. WHEN 用户选择票据并点击"彻底删除"，the system SHALL 显示二次确认对话框，IF 用户确认，the system SHALL 删除所有相关图片文件并永久移除票据数据。

#### REQ-111：自动清理提醒

**用户故事：** 作为系统，我需要在回收站票据即将被自动删除前提醒用户。

**验收标准：**

1. IF 回收站中的票据剩余可恢复天数小于等于 7 天，the system SHALL 在应用启动时显示本地通知，提醒用户及时处理。

---

## 6. 第四阶段需求：测试与基础发布

### 6.1 测试

#### REQ-120：功能边界测试

**用户故事：** 作为测试人员，我需要对应用进行全面的功能边界测试。

**验收标准：**

1. IF 网络中断，the system SHALL 显示"网络连接不可用，请检查网络设置"提示，使用本地缓存数据，并允许用户继续操作。
2. IF OCR 识别失败，the system SHALL 显示"识别失败，请重试"提示，并提供手动输入文字的入口，IF 用户重试 3 次仍失败，the system SHALL 建议用户检查图片质量。
3. IF 存储空间已满，the system SHALL 显示"存储空间不足，请清理缓存或删除部分票据"提示，并显示存储使用情况。
4. IF 扫描矫正处理异常，the system SHALL 显示"处理失败，已保留原图"提示，不修改原图。

#### REQ-121：边界输入测试

**用户故事：** 作为测试人员，我需要测试应用的输入边界。

**验收标准：**

1. THE system SHALL 验证标题不超过 100 字符、超出时截断并提示。
2. THE system SHALL 验证标签名称不超过 20 字符、超出时禁止输入。
3. THE system SHALL 验证合集名称不超过 30 字符、超出时禁止输入。
4. THE system SHALL 验证备注不超过 500 字符、超出时禁止输入。
5. THE system SHALL 验证图片选择不超过 9 张、超出时提示"最多选择 9 张图片"。

---

### 6.2 性能优化

#### REQ-130：性能要求

**用户故事：** 作为开发人员，我需要对应用进行性能优化。

**验收标准：**

1. THE system SHALL 确保应用冷启动时间不超过 3 秒。
2. THE system SHALL 确保 OCR 识别时间不超过 3 秒（正常网络条件下）。
3. THE system SHALL 确保列表滑动帧率保持在 60fps。
4. THE system SHALL 确保图片懒加载响应时间不超过 500ms。
5. THE system SHALL 确保搜索响应时间不超过 1 秒（1000 条数据以内）。

---

### 6.3 发布准备

#### REQ-140：发布准备

**用户故事：** 作为开发人员，我需要完成应用发布的准备工作。

**验收标准：**

1. WHEN 打包应用时，the system SHALL 配置应用图标（1024x1024 PNG）和启动页（ splash screen）。
2. WHEN 发布应用时，the system SHALL 提供完整的隐私政策文档，包含数据收集、数据使用、数据存储、数据共享、用户权利、儿童隐私、联系方式等章节。
3. WHEN 打包应用时，the system SHALL 支持 Android（APK、AAB）和 iOS（IPA）安装包格式。
4. THE system SHALL 提供应用商店的截图（手机截图 5 张、平板截图 5 张）和应用描述（不超过 4000 字符）。

## 7. 非功能性需求

### 7.1 性能需求

| 指标 | 要求 |
|------|------|
| 应用冷启动时间 | 不超过 3 秒 |
| OCR 识别时间 | 不超过 3 秒 |
| 列表滑动帧率 | 保持 60fps |
| 图片懒加载响应 | 不超过 500ms |
| 搜索响应时间 | 不超过 1 秒（1000 条数据） |

### 7.2 安全需求

| 需求 | 描述 |
|------|------|
| 数据存储 | 所有数据默认本地存储，不自动上传云端 |
| OCR 触发 | 必须由用户手动点击触发，不自动识别 |
| 登录验证 | 使用手机号验证码登录，验证码 5 分钟有效 |
| 登录限制 | 同一手机号 24 小时内最多发送 10 次验证码 |
| 设备限制 | 单设备 24 小时内最多尝试登录 20 次 |

### 7.3 兼容性需求

| 需求 | 描述 |
|------|------|
| Android 版本 | 支持 Android 6.0（API 23）及以上 |
| iOS 版本 | 支持 iOS 12 及以上 |
| 图片格式 | 支持 JPEG、PNG、WebP |
| 屏幕适配 | 支持主流手机屏幕比例（16:9、18:9、19.5:9、20:9） |

### 7.4 存储限制

| 资源 | 限制 |
|------|------|
| 单用户总存储容量 | 1 GB |
| 单张图片大小 | 不超过 20 MB |
| 单个票据图片数量 | 不超过 9 张 |
| 单用户票据数量 | 不超过 10000 条 |
| 标签数量 | 不限制 |
| 合集数量 | 不超过 100 个 |
